# Napkin.ai Migration Log

## Date: 2026-03-04

## Summary
Switched chart/graph generation from **Claude API (Anthropic)** to **Napkin.ai API**.
Charts are now rendered as SVG images generated by Napkin.ai instead of structured JSON rendered by React components.

## Changes Made

### 1. `api/generate-chart.js` (Vercel serverless function)
- **Before**: Called Claude API (`https://api.anthropic.com/v1/messages`) with a system prompt to generate structured JSON (flowchart, timeline, comparison, infographic, mindmap schemas). Returned JSON with `type`, `nodes`, `edges`, etc.
- **After**: Calls Napkin.ai API (`https://api.napkin.ai/v1/visual`) to generate SVG visuals. Uses async workflow: create request → poll status → download SVG → return as base64 data URL. Returns `{ type: "napkin-image", title, imageUrl }`.
- **Env var**: Now reads `NAPKIN_API_KEY` instead of `ANTHROPIC_API_KEY`.

### 2. `vite.config.js` (local dev middleware)
- **Before**: `/api/generate-chart` middleware called Claude API with same structured JSON approach.
- **After**: Same middleware now calls Napkin.ai API with create → poll → download flow.
- **Env var**: Now reads `NAPKIN_API_KEY` instead of `ANTHROPIC_API_KEY`.
- Added `'NAPKIN_'` to `envPrefix` array.

### 3. `src/services/chartAI.js` (frontend service)
- **Before**: `generateChart()` had 20s timeout.
- **After**: Timeout increased to 60s (Napkin async processing takes 5-15s + polling).

### 4. `src/components/ChartRouter.jsx` (chart router)
- **Before**: Only handled `flowchart`, `timeline`, `comparison`, `infographic`, `mindmap` types.
- **After**: Added `napkin-image` case that renders an `<img>` tag with the base64 SVG data URL.
- Existing renderers are **untouched** — they remain in code but won't be triggered while using Napkin.

### 5. `vercel.json` (Vercel config)
- Added `functions.maxDuration` config (60s for generate-chart and generate-sections, 30s for ask-question).
- Added `/api/(.*)` passthrough rewrite before catch-all SPA rewrite.

### 6. `.env.local`
- Added `NAPKIN_API_KEY=<token>` (user-provided).
- `ANTHROPIC_API_KEY` still needed for sections/Q&A features.

## What Was NOT Changed
- All CSS/styling
- Section generation (`api/generate-sections.js`, `useSectionGeneration.js`)
- Ask Q&A feature (`api/ask-question.js`, `askAI()`)
- All chart renderer components (FlowchartRenderer, TimelineRenderer, ComparisonRenderer, InfographicRenderer, MindmapRenderer)
- Component structure and layout
- `useChartGeneration.js` hook logic and caching

## How to Revert
To switch back to Claude API chart generation:
1. Restore `api/generate-chart.js` from git: `git checkout HEAD~1 -- api/generate-chart.js`
2. Restore the `/api/generate-chart` middleware in `vite.config.js` from git
3. Revert `chartAI.js` timeout from 60s back to 20s
4. Remove `napkin-image` case from `ChartRouter.jsx`
5. Remove `NAPKIN_API_KEY` from `.env.local`
6. Ensure `ANTHROPIC_API_KEY` has sufficient credits

## Update: 2026-03-04 — Dual Mode (Napkin + Claude Side-by-Side)

### Summary
Added Claude chart generation back alongside Napkin.ai. Both APIs run in parallel. Users can toggle between "Napkin.ai" and "Claude AI" in the Visual Breakdown panel.

### New Files Created
- `api/generate-chart-claude.js` — Vercel serverless function with original Claude chart generation logic
- `src/utils/chartCacheClaude.js` — Separate in-memory cache for Claude chart results
- `src/hooks/useMultiChartGenerationClaude.js` — Multi-chart hook using Claude API + Claude cache
- `src/hooks/useBackgroundPregenClaude.js` — Background pre-generation hook using Claude API

### Files Modified
- `src/services/chartAI.js` — Added `generateChartClaude()` function calling `/api/generate-chart-claude`. Both `generateChart` (Napkin) and `generateChartClaude` track `_generationTimeMs`.
- `vite.config.js` — Added `/api/generate-chart-claude` middleware handler with original Claude logic
- `src/pages/VisualizePage.jsx` — Added Claude hooks, `chartSource` toggle state (`'napkin'` | `'claude'`), toggle buttons in Visual Breakdown header, conditional rendering based on active source
- `src/components/ChartRouter.jsx` — Added "Generated in Xms" display at bottom of each chart
- `vercel.json` — Added `api/generate-chart-claude.js` with `maxDuration: 30`

### How the Toggle Works
- Both Napkin and Claude generation run in parallel on page load
- Each has its own cache (no cross-contamination)
- User clicks "Napkin.ai" or "Claude AI" toggle to switch the displayed chart
- Generation time (ms) shown at the bottom of each chart

## Napkin.ai API Details
- **Base URL**: `https://api.napkin.ai`
- **Auth**: `Authorization: Bearer <token>`
- **Create**: `POST /v1/visual` with `{ format, content, style_id, visual_query, number_of_visuals }`
- **Poll**: `GET /v1/visual/{id}/status`
- **Download**: Fetch from `generated_files[].url`
- **Style used**: Vibrant Strokes (`CDQPRVVJCSTPRBBCD5Q6AWR`)
- **Rate limit**: 60 requests/minute
- **File expiry**: 30 minutes (we convert to base64 inline, so no issue)
